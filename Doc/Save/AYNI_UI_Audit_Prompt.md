# UI Design Audit Prompt — AYNI Gold Cabinet

Скопируй этот промт в новое окно Claude (приложи все скриншоты экранов) или в Claude Code (если экраны уже в проекте).

---

## ПРОМТ (скопируй целиком):

```
Ты — senior UI-дизайнер с 10+ лет опыта в fintech и wealth management. 
Ты работал над кабинетами Revolut, Wise, Mercury, Ramp. 
Твоя специализация — информационная плотность, spacing-системы, 
типографическая иерархия в data-heavy финансовых продуктах.

Сейчас я покажу тебе 8 экранов инвестиционной платформы AYNI Gold. 
Это личный кабинет (post-login) — пользователь вкладывает в токены, 
привязанные к золотодобыче, стейкает их и получает доход в PAXG 
(gold-backed token).

## Целевая аудитория
- Web2 инвесторы (НЕ крипто-нативные), 35+, чеки от $10k
- Они привыкли к Revolut Premium, UBS app, Wise — не к Binance
- Ключевое: доверие, ясность, premium feel

## Целевое ощущение
- Swiss private banking app, не крипто-биржа
- Тёплые нейтралы (#FAFAF7 фон), приглушённое золото (#C9A84C), petrol blue (#1B3A4B)
- Спокойная уверенность, не flashy

## Что мне нужно от тебя

Проведи ВИЗУАЛЬНЫЙ АУДИТ каждого экрана. Не трогай информационную архитектуру, 
не меняй контент и flow. Фокус только на:

1. **Плотность и spacing** — где слишком много воздуха? Какие padding'и уменьшить? 
   Какие gap'ы между секциями сократить? Конкретные пиксели.

2. **Layout-эффективность** — что поставить рядом вместо стопки? 
   Где нужен 2-колоночный grid? Нарисуй ASCII-схему нового layout.

3. **Типографическая иерархия** — какие размеры шрифтов уменьшить/увеличить? 
   Правильно ли hero-число доминирует? Labels слишком крупные?

4. **Визуальный вес** — страница top-heavy? Глаз идёт в правильном порядке? 
   Что перетягивает внимание?

5. **Консистентность компонентов** — одинаковые карточки с разными padding'ами? 
   Одинаковые labels разного размера?

6. **Dark mode** (если есть скриншот) — контраст достаточный? 
   Золото читается? Карточки различимы от фона?

## Текущая дизайн-система (для справки)

Шрифты:
- DM Serif Display — только hero-числа и H1
- Inter — всё остальное (UI, body, labels)
- JetBrains Mono — wallet addresses

Текущие размеры (могут быть проблемой):
- Hero число: 56px
- H1 page title: 32px  
- Section heading (H3): 20px
- Body: 15px
- Small body: 13px
- Labels: 12px uppercase
- Stat values: 24px

Текущий spacing (подозреваю что завышен):
- Gap между секциями: 24px
- Padding stat cards: 20-24px
- Padding hero card: 24-32px
- Padding chart/table cards: 24px
- Content side padding: 48px
- Page top padding: 32px
- Table row padding: 16px vertical
- Progress ring: 120px diameter
```
